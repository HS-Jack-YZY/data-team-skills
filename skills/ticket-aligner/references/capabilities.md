## 三色标注约定

- 🟢 **可做** — 数据源稳定可达；工具组员熟悉；过去 90 天有过成功案例
- 🟡 **受限** — 数据源不全 / 凭证依赖外部 / 跨产品外推置信度受限 / 跨品类时基线失真
- 🔴 **不能做** — 缺授权 / 缺数据源 / 不在能力范围

skill 在生成对齐文档时**必须**与本表标注一致。如果某项能力在本表里**没有明确记录**，默认按 🟡 处理（不要替本文件拍 🟢），并在文档中提示"建议补充 capabilities.md"。

## 数据源 / 工具

### Amazon SP-API（自家 ASIN）
- 🟢 销量、广告报表、库存、Buybox、订单、流量
- 🟡 子级账号权限：部分子站点凭证需逐次申请
- 🔴 竞品数据（SP-API 仅覆盖自家店铺）
- **基线**
  - 单 ASIN 单市场销量拉取 ≈ **0.3h**
  - 全店报表（单市场） ≈ **1h**
  - 多市场全店汇总（5 市场）≈ **3h**

### Amazon AMC（Amazon Marketing Cloud）
- 🟢 ACoS / TACoS / CTR / CVR / CPC / 归因（默认 7d-click）
- 🟢 自家 Sponsored Products / Brands / Display 拆分
- 🟡 跨产品外推（例：用 BE3600 的 TACoS 推 BE10000）—— 仅作参考，不能等价
- 🔴 竞品 Sponsored Brands vs Products 拆分（缺广告主授权）
- **基线**
  - 1 条 SQL 查询（含取数 + 验证）≈ **1h**
  - 创意维度分析（按 ASIN × 创意）≈ **2h**

### Apify Reddit Scraper
- 🟢 子版块帖子 + 评论按关键词检索
- **基线**
  - 1 个关键词跑批（含清洗）≈ **1h**
  - 27 关键词批量成本 ≈ **$11**
- **约束**：每批最多 5 关键词避免限流；冷门关键词可能 0 结果

### Apify Amazon 评论（SHADER 代理 + Playwright）
- 🟢 单 ASIN 全量评论
- **基线**
  - 普通 ASIN（评论数 < 500）≈ **0.5h**
  - 热门 ASIN（如 MT6000，评论数 > 2000）≈ **2h**
- **约束**：包含在订阅中，无额外成本

### Keepa API
- 🟢 历史价格、销量估算（BSR-based）、BSR 趋势、库存信号
- **基线**
  - 单 ASIN 全字段 ≈ **0.2h**
  - 1 个市场 Top 100 BSR 全字段 ≈ **3h**
- **约束**：销量是估算值，绝对量级有 ±30% 误差，趋势方向可信

### Discourse API（GL.iNet 论坛 + OpenWrt 论坛）
- 🟢 关键词搜索 + 帖子 / 评论抓取
- **基线**
  - 1 个搜索词（含清洗）≈ **0.5h**

### SNBForums（Playwright MCP，无 API）
- 🟡 无 API，靠浏览器手动搜索
- **基线**
  - 1 个搜索词（含人工筛选）≈ **1.5h**

### Helium10
- 🟢 类目销量估算、关键词排名、ASIN 历史
- 🟡 跨平台数据：仅 Amazon 体系内
- **基线**
  - 单 ASIN 全字段 ≈ **0.2h**
  - 单关键词 + 关联 ASIN 池 ≈ **1h**

### 独立站后台（Shopify / 自建）
- 🟢 历史销售、订单、流量（GA4）、客户数据
- **基线**
  - 单市场全品类导出 + 清洗 ≈ **1h**
  - 多市场对比（5 市场）≈ **4h**

### 关键词词典标注
- 🟢 已沉淀 7 套词典：ID 设计 / 网口 / USB / 存储 / 屏幕 / Flint2 / 欧洲市场
- **基线**
  - 复用现有词典（应用到一批新评论） ≈ **0.5h**
  - 在现有词典上扩词 ≈ **2h**
  - 新建一套词典（首次构建）≈ **4h**

### LLM 语义分析（Claude）
- 🟢 评论情感、主题聚类、用户画像归类
- **基线**
  - 1000 条评论 ≈ **1h**（按 token 计费）
  - 人工校准（≥ 100 条标注集）≈ **2h**

### 问卷工具
- 🟡 当前可用工具有限，需评估（QuestionPro / Survio / 自建）
- 🔴 大样本投放（≥ 1000 份）需要预算审批
- **基线**
  - 设计问卷（5–8 题）≈ **2h**
  - 数据回收 + 清洗 ≈ **2h**（不含投放周期）

### 社交平台公开样本（IG / TikTok / YouTube）
- 🟡 仅公开数据可抓；曝光 / 互动数据置信度依赖平台 API 稳定性
- 🔴 竞品后台真实投放 ROI（无授权）
- **基线**
  - 单关键词 / 单 hashtag 抓样 ≈ **2h**

### 行业报告（IDC / Statista / Euromonitor）
- 🟡 部分数据需购买；公开数据延迟 6–12 个月
- **基线**
  - 1 份报告检索 + 摘要 ≈ **2h**
  - 跨多份报告交叉验证 ≈ **5h**

## 跨工具能力（成品级别）

### 11 维度全面情感分析
- 维度：WiFi 性能 / 稳定性 / 设置 / 性价比 / VPN / 固件 / 信号 / USB / 设计 / 端口 / 品牌力
- **基线**
  - 单品类全品牌 + 全维度热力图（关键词覆盖好）≈ **8h**
  - 跨市场（5 市场）×N（线性）

### 多市场竞品对标
- **基线**
  - 单市场单类目 5 竞品 ≈ **6h**
  - 多市场 ×N（线性）

### 销量预估模型
- **基线**
  - 单产品单市场基础模型 ≈ **4h**
  - 含广告 ROI 输入 ≈ **6h**
- 🟡 **未上市产品**（如 BE10000 / BE14000）置信度低，需明确告知 MVP 范围限制
- 🟡 缺乏自家产品历史数据的全新品类：销量预估置信度更低

### 价格分布分析
- **基线**
  - 单市场单类目 ≈ **2h**
  - 多市场（5 市场）≈ **6h**

### 用户画像归类（基于评论 / Reddit）
- **基线**
  - 单产品 ≈ **3h**（依赖关键词词典覆盖）
  - 单市场全品类 ≈ **6h**

### 报告撰写（HTML / Markdown）
- **基线**
  - 单章节（图表 + 解读）≈ **1.5h**
  - 整份报告（10 章左右）≈ **8h**
  - 应用 html-report 模板 ≈ **0.5h** 额外

## 促销活动数据（来自 Other Tickets.md 边界用例）

- 🟢 自家 BD / LD / DOTD / Coupon / Promotion Code 历史成效（SP-API + Amazon 后台）
- 🟡 行业基准（其他品牌的促销节奏 / 折扣力度）只能靠公开抽样
- **基线**
  - 单市场单促销类型全年盘点 ≈ **4h**
  - 5 市场对比 ≈ **12h**
  - 全年组合规划建议 ≈ **8h**

## 复盘 / 后验分析

- 🟢 自家活动后数据（订单 / 流量 / 广告 / 库存周转）
- 🟡 跨部门归因（KOL / 站外帖带流）依赖运营 / 市场组提供的活动埋点
- **约束**（重要）：**未在活动期预先打点的归因事后无法复原** —— skill 必须把这条作为"事前打点 vs 事后追溯"的硬性提示
- **基线**
  - 单一活动 5 市场全维度复盘（达成 / 归因 / 差异 / 改进）≈ **12h**
  - 整季度复盘 ≈ **20h**

## 跨品类适用性

> 本表所有基线**默认源自路由器品类**（Wi-Fi 路由器 / Travel Router / Mesh）。其他品类需校准。

- 🟡 **摄像头 / 安防设备**：评论关键词词典需新建（+ **4h**）；销量预估置信度受限于历史数据
- 🟡 **配件 / 周边**（线材 / 电源等）：评论稀疏，关键词词典 ROI 低；建议改用问卷或销售数据驱动
- 🟡 **完全新品类**（无自家历史数据）：销量预估默认置信度 **低**，必须在对齐文档中明确标注

**跨品类工时调整**：所有评论 / 词典 / 用户画像类工作 **× 1.5**；销量预估类标 🟡 + 替代方案。

## 已知不可达（🔴 清单 · 集中列出方便排除）

- 🔴 竞品 Sponsored Brands vs Products 拆分（缺广告主授权）
- 🔴 竞品 SP-API 任意数据（仅自家店铺有访问权）
- 🔴 第三方品牌真实 ROI / 利润率（无后台访问）
- 🔴 大样本问卷投放（≥ 1000 份）—— 需预算审批，不在常规能力内
- 🔴 历史未打点的跨部门归因事后追溯（如某次站外帖未埋活动 ID，事后无法定位转化来源）
- 🔴 Apify / Helium10 等订阅服务的非订阅功能（如某些超量调用需要升级）
